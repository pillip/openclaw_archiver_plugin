# Requirements: OpenClaw Archiver Plugin

## Goals (PRD 기반, 우선순위순)

| 순위 | 목표 | 설명 |
|------|------|------|
| G1 | 메세지 링크 저장/관리 | title + link 쌍으로 Slack 메세지 링크를 저장하고 프로젝트별로 분류/관리 |
| G2 | 사용자별 데이터 격리 | 각 사용자의 아카이브는 완전히 격리되어 다른 사용자가 열람 불가 |
| G3 | 결정론적 명령 파싱 | LLM bypass, 명령 파싱 기반으로 동일 입력에 항상 동일 출력 보장 |
| G4 | 대량 데이터 검색 성능 | 저장된 메세지가 많아져도 검색을 통해 빠르게 조회 가능 |

---

## Primary User

- OpenClaw가 설치된 Slack 워크스페이스의 팀원
- Slack 메세지 링크를 체계적으로 아카이빙하고 싶은 개인 사용자
- Slack DM을 통해 플러그인과 상호작용

---

## User Stories (우선순위: Must > Should > Could)

### Must

#### US-001: 메세지 저장
- **As a** Slack 사용자, **I want** Slack 메세지 링크를 title과 함께 저장하고 싶다, **so that** 나중에 빠르게 다시 찾을 수 있다.
- **우선순위**: Must
- **관련 FR**: FR-001, FR-002, FR-003, FR-004, FR-005
- **인수 조건**:
  - [ ] Given 사용자가 DM에서 `/archive save 회의록 https://slack.com/archives/C01/p123` 을 입력했을 때, When 명령이 처리되면, Then 해당 title("회의록")과 link가 사용자의 아카이브에 저장되고 성공 메시지가 반환된다.
  - [ ] Given 사용자가 `/archive save 회의록 https://slack.com/archives/C01/p123 /p Backend` 을 입력했을 때, When "Backend" 프로젝트가 존재하지 않으면, Then "Backend" 프로젝트가 자동 생성되고 메세지가 해당 프로젝트에 저장된다.
  - [ ] Given 사용자가 프로젝트를 지정하지 않았을 때, When 메세지를 저장하면, Then project_id가 NULL인 상태(미분류)로 저장된다.
  - [ ] Given 사용자가 title 또는 link를 누락했을 때, When 명령이 처리되면, Then 에러 메시지가 반환되고 저장되지 않는다.

#### US-002: 메세지 목록 조회
- **As a** Slack 사용자, **I want** 전체 또는 특정 프로젝트의 저장된 메세지 목록을 확인하고 싶다, **so that** 필요한 메세지를 찾을 수 있다.
- **우선순위**: Must
- **관련 FR**: FR-006, FR-007
- **인수 조건**:
  - [ ] Given 사용자가 `/archive list`를 입력했을 때, When 저장된 메세지가 있으면, Then 본인의 전체 아카이브 목록(id, title, link, project, created_at)이 반환된다.
  - [ ] Given 사용자가 `/archive list /p Backend`를 입력했을 때, When "Backend" 프로젝트에 메세지가 있으면, Then 해당 프로젝트의 메세지만 필터링되어 반환된다.
  - [ ] Given 사용자가 `/archive list`를 입력했을 때, When 저장된 메세지가 없으면, Then "저장된 메세지가 없습니다" 와 같은 빈 상태 메시지가 반환된다.
  - [ ] Given 사용자가 `/archive list /p NonExistent`를 입력했을 때, When 해당 프로젝트가 존재하지 않으면, Then 적절한 안내 메시지가 반환된다.

#### US-003: 메세지 검색
- **As a** Slack 사용자, **I want** 저장된 메세지를 title 키워드로 검색하고 싶다, **so that** 많은 메세지 중에서 빠르게 원하는 것을 찾을 수 있다.
- **우선순위**: Must
- **관련 FR**: FR-008, FR-009
- **인수 조건**:
  - [ ] Given 사용자가 `/archive search 회의록`을 입력했을 때, When title에 "회의록"이 포함된 메세지가 있으면, Then 매칭되는 메세지 목록이 반환된다.
  - [ ] Given 사용자가 `/archive search 회의록 /p Backend`를 입력했을 때, When 검색하면, Then "Backend" 프로젝트 범위 내에서만 검색 결과가 반환된다.
  - [ ] Given 사용자가 `/archive search 없는키워드`를 입력했을 때, When 매칭 결과가 없으면, Then "검색 결과가 없습니다" 와 같은 빈 상태 메시지가 반환된다.

#### US-004: 메세지 삭제
- **As a** Slack 사용자, **I want** 저장된 메세지를 삭제하고 싶다, **so that** 더 이상 필요 없는 아카이브를 정리할 수 있다.
- **우선순위**: Must
- **관련 FR**: FR-012, FR-013
- **인수 조건**:
  - [ ] Given 사용자가 `/archive remove 3`을 입력했을 때, When id=3인 메세지가 본인 소유이면, Then 해당 메세지가 삭제되고 성공 메시지가 반환된다.
  - [ ] Given 사용자가 `/archive remove 3`을 입력했을 때, When id=3인 메세지가 존재하지 않거나 본인 소유가 아니면, Then 에러 메시지가 반환되고 아무것도 삭제되지 않는다.

#### US-005: 개인 격리
- **As a** Slack 사용자, **I want** 내가 저장한 메세지를 다른 팀원이 볼 수 없도록 하고 싶다, **so that** 개인 아카이브의 프라이버시를 보장할 수 있다.
- **우선순위**: Must
- **관련 FR**: FR-018, FR-019
- **인수 조건**:
  - [ ] Given 사용자 A가 메세지를 저장했을 때, When 사용자 B가 `/archive list`를 실행하면, Then 사용자 A의 메세지는 결과에 포함되지 않는다.
  - [ ] Given 사용자 A가 id=5인 메세지를 소유하고 있을 때, When 사용자 B가 `/archive edit 5 새제목`을 실행하면, Then 에러 메시지가 반환되고 수정되지 않는다.
  - [ ] Given 사용자 A가 id=5인 메세지를 소유하고 있을 때, When 사용자 B가 `/archive remove 5`를 실행하면, Then 에러 메시지가 반환되고 삭제되지 않는다.

### Should

#### US-006: 타이틀 수정
- **As a** Slack 사용자, **I want** 저장된 메세지의 title을 수정하고 싶다, **so that** 더 나은 설명으로 업데이트할 수 있다.
- **우선순위**: Should
- **관련 FR**: FR-010, FR-011
- **인수 조건**:
  - [ ] Given 사용자가 `/archive edit 3 수정된 제목`을 입력했을 때, When id=3인 메세지가 본인 소유이면, Then title이 "수정된 제목"으로 변경되고 성공 메시지가 반환된다.
  - [ ] Given 사용자가 `/archive edit 3 수정된 제목`을 입력했을 때, When id=3인 메세지가 존재하지 않거나 본인 소유가 아니면, Then 에러 메시지가 반환되고 수정되지 않는다.
  - [ ] Given 사용자가 `/archive edit 3`을 입력했을 때(new_title 누락), When 명령이 처리되면, Then 사용법 안내 에러 메시지가 반환된다.

#### US-007: 프로젝트 관리
- **As a** Slack 사용자, **I want** 프로젝트 이름을 변경하거나 삭제하고 싶다, **so that** 프로젝트 구조를 정리할 수 있다.
- **우선순위**: Should
- **관련 FR**: FR-014, FR-015, FR-016, FR-017
- **인수 조건**:
  - [ ] Given 사용자가 `/archive project list`를 입력했을 때, When 본인의 프로젝트가 있으면, Then 프로젝트 목록(이름, 메세지 수)이 반환된다.
  - [ ] Given 사용자가 `/archive project rename BE Backend`를 입력했을 때, When "BE" 프로젝트가 본인 소유이고 "Backend"가 중복되지 않으면, Then 이름이 변경되고 성공 메시지가 반환된다.
  - [ ] Given 사용자가 `/archive project rename BE Backend`를 입력했을 때, When "Backend"라는 이름의 프로젝트가 이미 존재하면, Then 중복 에러 메시지가 반환되고 변경되지 않는다.
  - [ ] Given 사용자가 `/archive project delete Backend`를 입력했을 때, When "Backend" 프로젝트에 메세지가 있으면, Then 프로젝트가 삭제되고 해당 메세지들의 project_id가 NULL(미분류)로 변경된다.
  - [ ] Given 사용자가 `/archive project delete NonExistent`를 입력했을 때, When 해당 프로젝트가 존재하지 않으면, Then 에러 메시지가 반환된다.

### Could

- 해당 PRD에서 Could 수준으로 명시된 요구사항은 없음.

### Won't (이번 범위 제외)

- Slack 메세지 내용 크롤링/캐싱
- 메세지 공유 기능
- 태그/라벨 시스템
- 알림/리마인더 기능
- Slack 메세지 링크 유효성 검증

## Functional Requirements

### 메세지 저장 영역

#### FR-001: 메세지 저장 기본
- **설명**: `/archive save <title> <link>` 명령으로 Slack 메세지 링크를 title과 함께 저장한다.
- **우선순위**: Must
- **인수 조건**:
  - title과 link가 모두 제공되면 archives 테이블에 새 레코드가 생성된다.
  - 저장 시 현재 Slack user ID가 user_id 컬럼에 기록된다.
  - 저장 성공 시 생성된 archive의 id를 포함한 성공 메시지를 반환한다.
- **의존성**: FR-018 (데이터 격리)

#### FR-002: 프로젝트 지정 저장
- **설명**: `/archive save <title> <link> /p <project_name>` 형태로 프로젝트를 지정하여 저장할 수 있다.
- **우선순위**: Must
- **인수 조건**:
  - `/p <project_name>` 옵션이 포함되면 해당 프로젝트에 연결하여 저장한다.
  - 프로젝트 미지정 시 project_id는 NULL로 저장된다.
- **의존성**: FR-003

#### FR-003: 프로젝트 자동 생성
- **설명**: 저장 시 지정한 프로젝트가 존재하지 않으면 자동으로 생성한다.
- **우선순위**: Must
- **인수 조건**:
  - 해당 user_id + project_name 조합의 프로젝트가 없으면 projects 테이블에 새 레코드를 생성한 후 저장을 진행한다.
  - 이미 존재하는 프로젝트명이면 기존 프로젝트를 사용한다.
- **의존성**: 없음

#### FR-004: 저장 입력 검증
- **설명**: 필수 필드(title, link) 누락 시 에러를 반환한다.
- **우선순위**: Must
- **인수 조건**:
  - title이 비어있으면 에러 메시지를 반환하고 저장하지 않는다.
  - link가 비어있으면 에러 메시지를 반환하고 저장하지 않는다.
  - title만 있고 link가 없는 경우, 또는 link만 있고 title이 없는 경우 모두 에러 처리한다.
- **의존성**: 없음

#### FR-005: 저장 성공 응답
- **설명**: 저장 성공 시 저장된 메세지의 id, title, 프로젝트명을 포함한 확인 메시지를 반환한다.
- **우선순위**: Must
- **인수 조건**:
  - 응답에 생성된 archive id가 포함된다.
  - 프로젝트를 지정한 경우 프로젝트명이 응답에 포함된다.
- **의존성**: FR-001

### 메세지 조회 영역

#### FR-006: 전체 목록 조회
- **설명**: `/archive list` 명령으로 본인의 전체 아카이브 목록을 조회한다.
- **우선순위**: Must
- **인수 조건**:
  - 본인(user_id)의 모든 archive 레코드를 반환한다.
  - 각 항목에 id, title, link, 프로젝트명(없으면 미분류 표시), created_at을 포함한다.
  - 저장된 메세지가 없으면 빈 상태 안내 메시지를 반환한다.
- **의존성**: FR-018

#### FR-007: 프로젝트별 목록 조회
- **설명**: `/archive list /p <project_name>` 명령으로 특정 프로젝트의 메세지만 필터링하여 조회한다.
- **우선순위**: Must
- **인수 조건**:
  - 지정한 프로젝트에 속한 본인의 메세지만 반환한다.
  - 해당 프로젝트가 존재하지 않으면 적절한 안내 메시지를 반환한다.
  - 프로젝트는 존재하지만 메세지가 없으면 빈 상태 안내 메시지를 반환한다.
- **의존성**: FR-006, FR-018

### 메세지 검색 영역

#### FR-008: 키워드 검색
- **설명**: `/archive search <keyword>` 명령으로 title에서 키워드를 검색한다.
- **우선순위**: Must
- **인수 조건**:
  - title 컬럼에 대해 LIKE 패턴 매칭(부분 일치)을 수행한다.
  - 본인의 메세지만 검색 대상으로 한다.
  - 검색 결과가 없으면 빈 상태 안내 메시지를 반환한다.
  - keyword가 비어있으면 에러 메시지를 반환한다.
- **의존성**: FR-018

#### FR-009: 프로젝트 범위 검색
- **설명**: `/archive search <keyword> /p <project_name>` 명령으로 특정 프로젝트 범위 내에서 검색한다.
- **우선순위**: Must
- **인수 조건**:
  - 지정한 프로젝트 범위 내에서만 검색을 수행한다.
  - 해당 프로젝트가 존재하지 않으면 적절한 안내 메시지를 반환한다.
- **의존성**: FR-008

### 메세지 수정 영역

#### FR-010: 타이틀 수정
- **설명**: `/archive edit <id> <new_title>` 명령으로 저장된 메세지의 title을 수정한다.
- **우선순위**: Should
- **인수 조건**:
  - 본인 소유의 archive 레코드의 title을 new_title로 업데이트한다.
  - 수정 성공 시 확인 메시지를 반환한다.
  - id가 숫자가 아니면 에러 메시지를 반환한다.
  - new_title이 비어있으면 에러 메시지를 반환한다.
- **의존성**: FR-018

#### FR-011: 수정 권한 검증
- **설명**: 본인 소유가 아닌 메세지를 수정하려 하면 에러를 반환한다.
- **우선순위**: Should
- **인수 조건**:
  - 해당 id의 레코드가 존재하지 않거나 user_id가 다르면 동일한 에러 메시지를 반환한다 (존재 여부를 유추할 수 없도록).
  - 에러 시 어떠한 데이터도 변경되지 않는다.
- **의존성**: FR-018

### 메세지 삭제 영역

#### FR-012: 메세지 삭제
- **설명**: `/archive remove <id>` 명령으로 저장된 메세지를 삭제한다.
- **우선순위**: Must
- **인수 조건**:
  - 본인 소유의 archive 레코드를 삭제한다.
  - 삭제 성공 시 확인 메시지를 반환한다.
  - id가 숫자가 아니면 에러 메시지를 반환한다.
- **의존성**: FR-018

#### FR-013: 삭제 권한 검증
- **설명**: 본인 소유가 아닌 메세지를 삭제하려 하면 에러를 반환한다.
- **우선순위**: Must
- **인수 조건**:
  - 해당 id의 레코드가 존재하지 않거나 user_id가 다르면 동일한 에러 메시지를 반환한다 (존재 여부를 유추할 수 없도록).
  - 에러 시 어떠한 데이터도 삭제되지 않는다.
- **의존성**: FR-018

### 프로젝트 관리 영역

#### FR-014: 프로젝트 목록 조회
- **설명**: `/archive project list` 명령으로 본인의 프로젝트 목록을 조회한다.
- **우선순위**: Should
- **인수 조건**:
  - 본인의 모든 프로젝트를 반환한다.

#### FR-015: 프로젝트 이름 변경

- **우선순위**: Should

  - 변경 성공 시 확인 메시지를 반환한다.

## Out of Scope

다음 항목들은 이번 버전의 범위에 포함되지 않는다:

1. **Slack 메세지 내용 크롤링/캐싱** - 링크만 저장하며 메세지 본문은 저장하지 않는다.
2. **메세지 공유 기능** - 완전한 개인 격리 원칙에 따라 타인과의 공유는 지원하지 않는다.
3. **태그/라벨 시스템** - 프로젝트 분류만 지원한다.
4. **알림/리마인더 기능** - 저장된 메세지에 대한 알림을 발송하지 않는다.
5. **Slack 메세지 링크 유효성 검증** - 링크의 형식이나 접근 가능 여부를 검증하지 않는다.
6. **페이지네이션** - PRD에 언급되지 않음. 대량 목록에 대한 페이지 분할은 범위 외.
7. **메세지 프로젝트 이동** - 저장된 메세지를 다른 프로젝트로 이동하는 기능은 범위 외.
8. **벌크 작업** - 다건 삭제, 다건 저장 등 일괄 처리는 범위 외.
9. **인증/권한 관리** - Slack user ID를 신뢰하며 별도 인증 레이어를 두지 않는다.
10. **Block Kit UI** - PRD는 mrkdwn 텍스트 응답만 명시. Block Kit 메시지 형식은 범위 외.

| `cmd_save.py` | 응답에 `"        제목: ..."`, `"        프로젝트: ..."` 8칸 들여쓰기 | 볼드 레이블 + 일반 텍스트로 교체 |

| `cmd_remove.py` | 응답에 `"        {title}"` 8칸 들여쓰기 | 볼드 서식 적용, 들여쓰기 제거 |
| `cmd_project_list.py` | 목록 항목에 `"        {name}     {count}건"` 8칸 들여쓰기 | 볼드 프로젝트명, 들여쓰기 제거 |
| `cmd_project_rename.py` | 응답에 `"        {old} -> {new}"` 8칸 들여쓰기 | 볼드 서식 적용, 들여쓰기 제거 |
